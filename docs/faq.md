# 云端版 FAQ

## 1. 这个项目和官方 Open-AutoGLM 是什么关系？

官方 Open-AutoGLM 提供 Phone Agent 核心能力。本项目把它封装成云端 FastAPI 服务，并提供手机 App、任务系统、Trace 展示和远程 ADB 部署方案。

## 2. 运行时本地电脑必须开着吗？

不需要。云端版的后端和 Agent 都运行在云服务器上。电脑只在部署、维护或重新开启手机 TCP ADB 时可能需要。

## 3. 云服务器必须开吗？

必须。云端版依赖云服务器运行 Docker 容器。

## 4. REAL 模式显示无设备怎么办？

检查：

1. 手机无线调试是否开启。
2. 手机 Tailscale 是否在线。
3. `ADB_CONNECT_ADDRESS` 是否填写正确。
4. 容器内执行 `adb devices` 是否能看到设备。
5. 手机是否弹出过 ADB 授权确认。

## 5. 为什么任务已经打开 App，但结果显示失败？

部分任务会在 Agent 正式运行前由 server 做预启动或快路径操作。如果后续 Agent 因截图超时、API 失败、设备断连等问题失败，最终状态仍可能是 failed。

## 6. 为什么小红书比美团慢？

美团搜索做了专项优化，小红书主要走通用 Agent 链路。通用链路需要多轮“截图 → 模型理解 → ADB 执行 → 再截图确认”，而云端远程 ADB 还会增加截图传输延迟。

## 7. API Key 可以提交到 GitHub 吗？

不可以。只能提交 `.env.example` 和 `.env.cloud.example`，真实 `.env.cloud` 必须被 `.gitignore` 排除。

## 8. 视频要直接放仓库吗？

小视频可以放在 `demo/`。如果视频较大，建议放 GitHub Release 或网盘，再在 README 中放链接。

## 9. 当前版本的基准是什么？

当前云端版以“云服务器远程控制真实手机，并完成美团搜索类任务”为冻结基准。
