# 作业对照与面试官导读（Cloud 云端版）

## 一、项目定位说明（重要）

**本仓库是个人加分扩展，不是 Open-AutoGLM 作业的硬性要求。**

官方作业核心是：Python + ADB + 智谱 API + Phone Agent 跑通真实手机控制。  
USB v1 / WiFi v1 已完整覆盖作业要求；云端版解决的是：

> **后端迁到云服务器，手机跨网络远程连接，本地电脑不用一直开着。**

这对 AI 应用落地 / 工程化岗位是加分项，面试时可作为「进阶能力」单独讲。

---

## 二、与 Open-AutoGLM 官方作业的对照

| 官方要求 | 云端版如何实现 |
| --- | --- |
| phone_agent 内核（截图→VLM→ADB） | ✅ 完整保留 |
| 远程 ADB 调试 | ✅ Tailscale + 无线 ADB，比同 WiFi 更远 |
| 智谱 BigModel API | ✅ 云容器内调用 |
| 自然语言控制手机 | ✅ App → 公网 → FastAPI → Agent |
| **作业未要求** Docker 云端部署 | ⭐ 本仓库重点 |
| **作业未要求** 手机 App + Trace | ⭐ 与 USB/WiFi 版共用 |

## 四、30 秒项目介绍（面试可直接背）

> 在 USB 和 WiFi 本地版跑通后，我把后端部署到云服务器 Docker 里，手机通过 Tailscale 和无线 ADB 让云端容器能远程控制真实手机。用户 App 填公网 IP 就能发任务，电脑不用一直开着。我还处理了远程截图慢、Take_over 非交互、美团搜索演示稳定性等云端特有问题。这是作业之上的工程扩展，不是官方 CLI 的必做项。

## 五、面试官常问 5 题

**Q1：为什么作业不做云版也够了？**

> 官方要求是跑通 Phone Agent。USB/WiFi 版已满足。云版是我主动做的产品化升级。

**Q2：云端怎么连上手机的？**

> 手机和云服务器都装 Tailscale 进同一虚拟网，手机开无线调试，容器里 `adb connect 手机TailscaleIP:5555`。

**Q3：远程和本地最大区别？**

> 截图经 Tailscale 传输更慢，容易超时；subprocess 无 TTY 时 Take_over 会 EOF，我做了自动跳过；还有键盘检查、截图压缩等补丁。

**Q4：为什么美团比小红书快？**

> 美团做了搜索快路径和 UI 树辅助；小红书走通用多轮 Agent，远程截图延迟会累加。

**Q5：生产能用吗？**

> 当前是 demo / 作品集级别。后续可加 HTTPS、鉴权、WebSocket、多设备队列等。
